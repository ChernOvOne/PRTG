# 🌐 WEB SERVER MANAGER — `webm`

Скрипт автоматической настройки VPS под статический сайт с Nginx, SSL, Fail2ban и SSI-конфигом.  
При создании сайта `index.html` и `config.js` скачиваются с GitHub автоматически.

---

## ⚡ Быстрый старт — одна команда

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ChernOvOne/PRTG/main/webserver-setup.sh)
```

После установки управляй сервером:

```bash
sudo webm
```

## Требования

- **Ubuntu** 20.04 / 22.04 / 24.04
- Доступ по SSH под **root** или пользователем с `sudo`
- Домен с A-записью → IP сервера (нужно для SSL)
- Порты **22, 80, 443** доступны (скрипт настроит UFW сам)

---

## Что делает установка

Запускаешь одну команду — скрипт проходит все шаги с живым выводом в терминал:

| Шаг | Что происходит |
|-----|---------------|
| `apt update` | Обновление списков пакетов |
| `apt upgrade` | Обновление системы (видишь каждый пакет) |
| Базовые утилиты | curl, wget, git, unzip, build-essential, gnupg... |
| **Nginx** | Установка и запуск веб-сервера |
| **Certbot** | Для SSL-сертификатов Let's Encrypt |
| **UFW** | Файрвол: открыты только 22, 80, 443 |
| **Fail2ban** | Защита SSH и Nginx от брутфорса (бан после 5 попыток за 10 мин) |
| Авто-обновления | Патчи безопасности устанавливаются автоматически каждый день |
| SSL auto-renew | Cron: `certbot renew` запускается ежедневно в 03:00 |
| Безопасность Nginx | Скрыта версия, заголовки XSS/HSTS/Frame/MIME, rate limiting |
| **Команда `webm`** | Скрипт копируется в `/usr/local/bin/webm` — запускается из любого места |
| **Файлы с GitHub** | `index.html` и `config.js` скачиваются в кэш `/var/www/_github_cache/` |

> Полный лог установки всегда доступен: `tail -f /var/log/webm.log`

## Создание сайта

```
sudo webm → 3) Создать новый сайт
```

Скрипт задаст три вопроса:

```
Домен (например: example.com):    hideyou.top
Ссылка на Telegram-бота VPN:      https://t.me/HideYouTop_bot
Ссылка на поддержку Telegram:     https://t.me/Hide_You
```

> Если нажать Enter — подставятся заглушки, которые можно поменять позже в `config.js`.

**Что происходит автоматически:**

1. Создаётся директория сайта `/var/www/ДОМЕН/html/`
2. `index.html` скачивается с GitHub (или берётся из кэша `/var/www/_github_cache/`) и в нём прописывается путь к `config.js` конкретного домена
3. `config.js` скачивается с GitHub, в него подставляются введённые ссылки на бота и поддержку, и кладётся в `/etc/nginx/private/ДОМЕН/config.js`
4. Настраивается Nginx: SSI включён, прямой запрос `config.js` через браузер возвращает 404
5. Предлагается сразу выпустить SSL-сертификат

После создания — добавь свои прокси-серверы:

```bash
nano /etc/nginx/private/ДОМЕН/config.js
```

## Настройка config.js

Файл хранится **только на сервере** в `/etc/nginx/private/ДОМЕН/config.js`.  
Браузер не может его запросить напрямую — Nginx вставляет содержимое в HTML через SSI.

### Открыть редактор

```bash
nano /etc/nginx/private/ДОМЕН/config.js
```

### Структура файла

```javascript
const CFG = {

  // ── ССЫЛКИ ────────────────────────────────────────────────
  botLink:     "https://t.me/HideYouTop_bot",   // ← Telegram-бот VPN
  supportLink: "https://t.me/Hide_You",          // ← поддержка

  // ── MTProto ПРОКСИ ────────────────────────────────────────
  // Каждый объект = одна карточка на странице
  mtproto: [
    {
      name: "ПРОКСИ RU-1",
      tag:  "ОСНОВНОЙ",
      addr: "tgrp.hideyou.top:8443",             // отображаемый адрес
      link: "tg://proxy?server=tgrp.hideyou.top&port=8443&secret=СЕКРЕТ",
      desc: "Основной · Москва"
    },
    // добавляй сколько угодно объектов
  ],

  // ── SOCKS5 ПРОКСИ ─────────────────────────────────────────
  // login и pass — пустая строка "" если авторизация не нужна
  socks5: [
    {
      name:  "SOCKS5 RU-1",
      tag:   "РОССИЯ",
      ip:    "185.147.26.8",
      port:  "1080",
      login: "hide",
      pass:  "пароль",
      desc:  "Москва · для Telegram и браузера"
    },
  ],

  // ── ТЕКСТЫ СТРАНИЦЫ ───────────────────────────────────────
  t: {
    heroLine1:  "TELEGRAM",
    heroLine2:  "РАЗБЛОКИРОВАН",
    heroLine3:  "В 2 КЛИКА И БЕСПЛАТНО",
    heroSub:    "Бесплатный прокси для Telegram...",
    footerCopy: "© 2025 HIDEYOU · ALL SYSTEMS NOMINAL"
    // ... остальные тексты
  }
};
```

### Загрузить с локального компьютера

```bash
scp config.js root@ВАШ_IP:/etc/nginx/private/ДОМЕН/config.js
```

> После редактирования файла перезагружать Nginx **не нужно** — SSI читает файл при каждом запросе.

## SSL-сертификаты

### Выпустить SSL (при создании сайта или отдельно)

```
sudo webm → 6) Выпустить SSL для домена
```

Скрипт спросит домен и email → запустит Certbot → SSL выпускается сразу для `домен` и `www.домен`.

> **Важно:** до запуска A-запись домена должна указывать на IP сервера.

### Обновить вручную

```
sudo webm → 7) Обновить все SSL вручную
```

### Авто-обновление

Настраивается автоматически при установке.  
Cron запускает обновление каждый день в **03:00**:

```bash
certbot renew --quiet --nginx && systemctl reload nginx
```

Проверить:
```bash
crontab -l | grep certbot
# или через systemd:
systemctl list-timers | grep certbot
```

## Структура файлов на сервере

```
/usr/local/bin/webm                        ← команда webm (сам скрипт)
/var/log/webm.log                          ← лог всех действий скрипта

/var/www/ДОМЕН/html/
  └── index.html                           ← страница сайта

/etc/nginx/
  ├── sites-available/ДОМЕН               ← конфиг сайта
  ├── sites-enabled/ДОМЕН                 ← симлинк (активный сайт)
  ├── conf.d/security.conf                ← заголовки безопасности, rate limiting
  └── private/ДОМЕН/
        └── config.js                     ← ⚠️ приватный конфиг (прокси, пароли)
                                             недоступен из браузера — только SSI

/var/www/_github_cache/
  ├── index.html                           ← кэш с GitHub (скачивается при установке)
  └── config.js                           ← кэш шаблона с GitHub

/etc/fail2ban/jail.local                   ← настройки Fail2ban
/etc/apt/apt.conf.d/50unattended-upgrades  ← авто-патчи
```

### Как SSI скрывает config.js

Nginx получает запрос → читает `index.html` → находит SSI-директиву → вставляет содержимое `config.js` прямо в HTML → отдаёт браузеру уже готовую страницу. Отдельного HTTP-запроса к `config.js` не существует.

```
Браузер                    Nginx                    Диск
  │── GET /index.html ──▶ │                          │
  │                        │── читает index.html ──▶ │
  │                        │── читает config.js ───▶ │  (SSI)
  │◀── готовый HTML ───── │
  │
  │── GET /config.js ───▶ │── 404 (заблокировано) ─▶│
```

## Обновление index.html и config.js

### Обновить index.html на сайте

Внёс правки в `index.html` и запушил на GitHub:

```bash
# Обновить кэш и подтянуть на сайт
curl -fsSL "$GITHUB_RAW/index.html" -o /var/www/ДОМЕН/html/index.html
# Подставить домен в SSI-тег если нужно
sed -i "s|ВАШ_ДОМЕН|ДОМЕН|g" /var/www/ДОМЕН/html/index.html
```

Или загрузить напрямую с компьютера:

```bash
scp index.html root@ВАШ_IP:/var/www/ДОМЕН/html/index.html
```

### Обновить config.js (прокси, пароли, ссылки)

```bash
# Прямо на сервере
nano /etc/nginx/private/ДОМЕН/config.js

# Или загрузить с компьютера
scp config.js root@ВАШ_IP:/etc/nginx/private/ДОМЕН/config.js
```

> Перезагружать Nginx после правки `config.js` **не нужно**.

### Обновить сам скрипт webm

```bash
curl -fsSL https://raw.githubusercontent.com/ChernOvOne/PRTG/main/webserver-setup.sh \
  -o /usr/local/bin/webm && chmod +x /usr/local/bin/webm && echo "✅ webm обновлён"
```
